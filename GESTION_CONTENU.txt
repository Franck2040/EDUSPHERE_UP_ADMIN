
# GESTION DU CONTENU - DOCUMENTATION COMPLÈTE

## FICHIERS PRINCIPAUX ET LEURS RÔLES

### Pages principales
- `src/pages/AdminContent.tsx`
  * Rôle: Page principale de gestion du contenu
  * Fonctions: Navigation entre livres/examens, stats globales
  * Importance: CRITIQUE - Hub central du contenu

### Gestion des livres
- `src/components/admin/content/BookManager.tsx` (276 lignes)
  * Rôle: Gestion complète des livres
  * Fonctions: CRUD livres, aperçu PDF, téléchargement
  * Importance: CRITIQUE - Composant principal livres

- `src/components/admin/content/BookForm.tsx`
  * Rôle: Formulaire de création/modification de livres
  * Fonctions: Upload fichier, métadonnées, validation
  * Importance: HAUTE - Interface de saisie

### Gestion des examens
- `src/components/admin/content/ExamManager.tsx` (260 lignes)
  * Rôle: Gestion complète des examens
  * Fonctions: CRUD examens, aperçu PDF, téléchargement
  * Importance: CRITIQUE - Composant principal examens

- `src/components/admin/content/ExamForm.tsx`
  * Rôle: Formulaire de création/modification d'examens
  * Fonctions: Upload fichier, liaison écoles/niveaux
  * Importance: HAUTE - Interface de saisie

### Composants utilitaires
- `src/components/admin/content/ContentStats.tsx` (214 lignes)
  * Rôle: Statistiques du contenu
  * Fonctions: Métriques livres/examens, tendances
  * Importance: MOYENNE - Dashboard

- `src/components/admin/content/PDFPreviewModal.tsx`
  * Rôle: Aperçu des fichiers PDF
  * Fonctions: Visualisation documents, navigation
  * Importance: MOYENNE - UX améliorée

- `src/components/admin/content/ContentAnalytics.tsx`
  * Rôle: Analytics avancées du contenu
  * Fonctions: Graphiques, analyses usage
  * Importance: BASSE - Fonctionnalité avancée

### Composants partagés
- `src/components/admin/content/FileUpload.tsx`
  * Rôle: Composant d'upload de fichiers
  * Fonctions: Drag & drop, validation, progress
  * Importance: HAUTE - Réutilisable

## TABLES DATABASE LIÉES
- `books` - Données des livres
- `exams` - Données des examens
- `book_ratings` - Évaluations des livres
- `schools` - Écoles (pour examens)
- `academic_levels` - Niveaux académiques
- `user_history` - Historique d'activité contenu

## POUR MODIFIER/AJOUTER DES FONCTIONNALITÉS

### Ajouter un nouveau type de contenu:
1. Créer: `src/components/admin/content/[NouveauType]Manager.tsx`
2. Créer: `src/components/admin/content/[NouveauType]Form.tsx`
3. Modifier: `src/pages/AdminContent.tsx` (ajouter onglet)
4. Créer: Table database correspondante

### Ajouter un nouveau champ à un livre:
1. Modifier: `src/components/admin/content/BookForm.tsx`
2. Modifier: Table `books` dans database
3. Modifier: `src/components/admin/content/BookManager.tsx` (affichage)

### Améliorer les statistiques:
1. Modifier: `src/components/admin/content/ContentStats.tsx`
2. Ajouter: Nouvelles requêtes database
3. Possiblement modifier: `ContentAnalytics.tsx`

## PROBLÈMES ACTUELS ET SOLUTIONS

### Problème: Fichiers trop longs
- `BookManager.tsx` (276 lignes) - BESOIN REFACTORING
- `ExamManager.tsx` (260 lignes) - BESOIN REFACTORING
- `ContentStats.tsx` (214 lignes) - BESOIN REFACTORING

### Solution: Séparer en composants
- Extraire table en composant `ContentTable.tsx`
- Créer hooks pour la logique métier
- Séparer actions en composants dédiés

### Problème: Code dupliqué
- BookManager et ExamManager très similaires
- Solution: Créer composant générique `ContentManager.tsx`
